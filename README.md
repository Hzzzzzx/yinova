# Yinova

64-hexagram control panel · openclaw multi-instance · one-click deploy · v1.1

> Manage openclaw instances via a web panel (Yin main gateway + 64 hexagram nodes). Supports mainstream AI providers—fill in API Key and go.

## Quick Start

```bash
# 1. Clone and install
git clone https://github.com/goldct/yinova && cd yinova
chmod +x install.sh start-panel.sh
./install.sh

# 2. Start panel
./start-panel.sh
# Or on Mac: double-click start-panel.command (opens browser automatically)
```

3. Open http://localhost:3999

4. **First run**: You'll be redirected to config page → choose provider → fill API Key → save. Yin and 64 hexagrams will use it immediately; no restart needed.

## Structure

```
yinova/
├── openclaw/          # Fetched by install, not in repo
├── hex-template/     # Hexagram template
├── hexes/             # Generated by install, 64 hexagrams (not in repo)
├── yin/               # Main gateway (port 18789)
├── panel-web/         # Web panel (port 3999)
├── install.sh         # First-time install
├── start-panel.sh     # Start panel
└── start-panel.command # Mac double-click to start
```

## Portable Use

After `install` + `npm install`, copy the whole `yinova` directory to a USB or another machine. Target machine only needs Node.js 18+; run `./start-panel.sh` or double-click `start-panel.command`.

## FAQ

- **Yin unreachable / returns HTML instead of JSON**: 1) Click "Yin · Start" first; 2) Ensure `yin/moltbot.json` has `gateway.http.endpoints.chatCompletions.enabled = true` (install sets this); 3) After copying project, run `./install.sh` again to fix paths (including workspace in molbot.json), token, and config.
- **Port in use**: Defaults: panel 3999, main gateway 18789, hexagrams 18791+. On config page, click "Check ports"—if any are in use, the panel auto-finds free ports and updates config.json, workers.conf, and moltbot.json. No manual editing needed.
- **No API key / API call failed**: Choose provider on config page, fill API Key, save. It syncs to Yin and hexagrams; restart Yin/hex if needed.
- **Empty model list**: Ensure `./install.sh` completed (openclaw must be fetched).
- **Install failed**: Ensure Node.js 18+ and npm are installed; check network (GitHub and npm registry).
- **Hex shows "running" without clicking start**: Panel detects by port. If a previous terminal left a process, the port stays in use. Click "Stop" on that hex, or "Close all terminals".
