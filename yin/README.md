# Yin — 64 Hexagram Main Gateway

Yin is Yinova's main gateway node (port 18789), managing 64 hexagram sub-nodes.

## Start

```bash
# After install and panel start, click "Yin · Start" on panel
# Or manually:
cd <Yinova root>/yin
./start-yin.sh
```

> `start-yin.sh` will:  
> 1. Start main gateway (port 18789)  
> 2. Launch openclaw TUI

## Config

`yin/moltbot.json` is generated by `install.sh` from `yin/moltbot.json.example`, **not in repo**.  
For manual edits, see `yin/moltbot.json.example`.

Key fields:

| Field | Description |
|-------|-------------|
| `gateway.port` | Main gateway port, default 18789 |
| `gateway.auth.token` | Random by install.sh |
| `agents.defaults.model.primary` | Default primary model |

## Capabilities

Yin uses `workspace/skills/panel-control/` to operate panel API from chat:

- Query hex status (`GET /api/robots`)
- Create projects (`POST /api/projects`)
- Assign hexagrams to projects (`PUT /api/projects/:id`)
- Start/stop hexagrams (`POST /api/open-tui/:id` / `POST /api/stop/:id`)

## Structure

```
│   ├── SOUL.md           # Principles
│   └── skills/
│       └── panel-control/
│           └── SKILL.md  # Panel control skill

```

## Troubleshooting

**No API key found**  
Config page save writes to `yin/agents/main/agent/auth-profiles.json`.  
If still failing, check file exists or re-save on config page.

**Port 18789 in use**  
Use "Check ports" on config page, or:
```bash
lsof -i :18789
kill -9 <PID>
```
